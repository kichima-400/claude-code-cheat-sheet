🔗 フック設定 (Hooks)
フックイベント一覧
SessionStartセッション開始・再開・クリア時
UserPromptSubmitユーザープロンプト送信時
PreToolUseツール実行前（ブロック可）
PermissionRequestパーミッション確認時
PermissionDeniedオートモードで拒否時
PostToolUseツール実行成功後
PostToolUseFailureツール実行失敗後
Notification通知送信時
StopClaude が応答終了時
StopFailureAPIエラーで終了時
SubagentStart / Stopサブエージェント起動・終了時
TaskCreated / Completedタスク作成・完了時
PreCompact / PostCompactコンテキスト圧縮前後
FileChanged監視ファイル変更時
WorktreeCreate / RemoveWorktree 作成・削除時
InstructionsLoadedCLAUDE.md 読み込み時
ConfigChange設定ファイル変更時
CwdChanged作業ディレクトリ変更時
SessionEndセッション終了時
ElicitationMCP ツール入力要求時
フックタイプ
type: "command"シェルコマンドを実行
type: "http"外部エンドポイントに HTTP POST
type: "prompt"LLM で単一ターン評価
type: "agent"マルチターン検証（ツール使用可）
終了コード (command フック)
exit 0操作を許可（stdout は構造化出力として解析）
exit 2操作をブロック（stderr を Claude へフィードバック）
exit その他操作を許可（stderr はログのみ）
設定例（PostToolUse + Prettier）
{ "hooks": { "PostToolUse": [{ "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "npx prettier --write \ $(jq -r '.tool_input.file_path')" }] }] } }

スキルファイルの配置場所
~/.claude/skills/<n>/個人スキル（全プロジェクト）
.claude/skills/<n>/プロジェクトスキル
<plugin>/skills/<n>/プラグインスキル
各フォルダに SKILL.md を置く
SKILL.md frontmatter
--- name: my-skill description: スキルの説明・使用場面 argument-hint: [arg1] [arg2] allowed-tools: Read, Grep model: claude-sonnet-4-6 effort: high context: fork # サブエージェントで実行 agent: Explore # エージェント型 user-invocable: false # Claudeのみ呼び出し可 ---

$ARGUMENTSすべての引数

🔐 パーミッションモードとモデル
パーミッションモード（Shift+Tab で循環）
default各アクションを確認ダイアログで許可/拒否
